# Threat Analysis Data Collection Report – Equifax Inc.

## Course  
CYB/545 – Threat Intelligence

## Week 4 Project  
Threat Analysis Data Collection Report

---

## 1. Data-Collection Criteria

**Definition**  
Data-collection criteria are structured guidelines used to ensure that information gathered about potential threats to Equifax is relevant, timely, and actionable.

**Factors Considered**  
- **Relevance**: Data must address specific or emerging threats to Equifax  
- **Timeliness**: Information should reflect the current threat environment  
- **Source Credibility**: Only reputable, vetted sources are used  
- **Completeness**: Data must provide enough detail to inform actionable insights  
- **Legal and Ethical Standards**: Collection practices must adhere to applicable laws and organizational policies  

---

## 2. Types of Data Needed

- **Threat Intelligence Reports**: Targeted analyses of vulnerabilities and threats  
- **Incident Reports**: Documentation from past breaches and security events  
- **Vulnerability Assessments**: System weaknesses that may be exploited  
- **Attack Vectors**: Common techniques used by threat actors against Equifax  
- **Environmental Data**: Industry context and operational surroundings  

---

## 3. Criteria for Reliability

- **Source Verification**: Cross-referencing with multiple sources  
- **Consistency**: Alignment with known intelligence and threat patterns  
- **Historical Accuracy**: Proven track record of source reliability  
- **Peer Review**: Review by cybersecurity experts for data validation  

---

## 4. Operational Security (OPSEC) Limitations

- **Information Sensitivity**: Some data may introduce risk if exposed  
- **Access Controls**: Limiting access to sensitive internal datasets  
- **Communication Channels**: Risk of data interception if not secured  
- **Insider Threats**: Potential for internal misuse of sensitive information  

---

## 5. Strategies, Methods, Tools, Sources, and Storage

**Strategies**  
- **OSINT** (Open Source Intelligence)  
- **HUMINT** (Human Intelligence via interviews)  
- **TECHINT** (Technical Intelligence via system-level data collection)  

**Methods**  
- Surveys and stakeholder interviews  
- Automated scraping of public intelligence feeds  
- Integration of security telemetry through SIEM  

**Tools**  
- SIEM platforms (e.g., Splunk, QRadar)  
- Threat Intelligence Platforms (TIPs)  
- Database management and secure cloud repositories  

**Sources**  
- Government threat advisories  
- Industry-specific cybersecurity publications  
- Cybersecurity blogs, vendor whitepapers, and dark web monitoring  

**Storage**  
- Encrypted cloud environments with MFA access  
- Segmented, locally stored databases with automated backups  

---

## 6. Data Normalization and Alignment with Intelligence Goals

- **Standardization**: Ensure all incoming data fits a common schema  
- **Integration**: Enable merging of multi-source intelligence for correlation  
- **Query Optimization**: Structure data to support efficient, real-time analytics  

These processes allow threat analysts to align raw data with strategic intelligence objectives efficiently.

---

## 7. Methods and Tools for Validation and Analysis

**Validation Methods**  
- Cross-validation of datasets  
- Statistical trend analysis  

**Analysis Tools**  
- Data visualization platforms (e.g., Tableau, Power BI)  
- Machine learning for anomaly and behavior-based detection
