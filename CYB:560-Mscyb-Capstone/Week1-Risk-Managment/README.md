---

## Threat Response Playbooks

### Playbook 1: SQL Injection  
- **Preparation**: Developer training, WAF implementation  
- **Detection**: Automated scanning, log monitoring  
- **Containment**: IP blocking, app disabling  
- **Eradication**: Code fixes, DBMS patching  
- **Recovery**: Data restoration, validation  
- **Post-Incident**: Documentation and policy updates  

### Playbook 2: Website Compromise  
- **Preparation**: Backups, MFA, IR planning  
- **Detection**: Anomaly detection via traffic analysis  
- **Containment**: Take site offline, isolate servers  
- **Eradication**: Remove malware, restore from clean backup  
- **Recovery**: Enhanced monitoring and stakeholder communication  
- **Post-Incident**: Review and tighten attack surface  

### Playbook 3: Exploitation of Windows 7 Workstations  
- **Preparation**: Migration plans, awareness training  
- **Detection**: Monitor for unusual workstation behavior  
- **Containment**: Network isolation, disable access  
- **Eradication**: Malware cleanup, residual patching  
- **Recovery**: System imaging, restore operations  
- **Post-Incident**: Full documentation and transition timeline  

---

## NIST Cybersecurity Framework Alignment  
- Adopted to support risk-based decision-making  
- Framework: Identify → Protect → Detect → Respond → Recover  
- Benefits: Scalable, widely accepted, supports compliance  
- Compared to: FFIEC CAT, ISO 27001 — more adaptable and holistic
